LENS: Exploring and Exploiting Security Vulnerabilities in Self-Hosted LLM Services

Zhihuang Liu1, Ling Hu1, Yonghao Tang1, Tongqing Zhou1, Fang Liu2, Zhiping Cai1*
1National University of Defense Technology, 2Hunan University
*Corresponding Author
Paper Code & Knowledge Base

LENS Framework: The first framework for systematic exploration and automated exploitation of security vulnerabilities in self-hosted LLM services.

Abstract

English: In response to the recent popular trend of open-source large language model (LLM) deployment, this paper proposes the first framework for systematically exploring and automatically exploiting security vulnerabilities in self-hosted LLM services. We conducted a large-scale empirical assessment on 16 mainstream LLM deployment platforms and over 70,000 deployment targets, filling the research gap regarding the actual severity of vulnerabilities in LLM deployments. We also proposed corresponding security defense techniques. This research has been assigned multiple high-severity (or above) CVE IDs.

Chinese: 面向近来流行的开源大模型部署趋势，该论文提出了首个系统性探索和自动化渗透利用自部署大模型服务安全漏洞的框架，对16个主流的大模型部署平台、7万多个部署目标进行了大规模实证评估，填补了关于大模型部署脆弱性实际危害程度的研究空白，并提出了对应的安全防御技术。该研究获得了多个高危级别以上的漏洞编号。

Discovered Vulnerabilities (CVEs)

Our research uncovered significant security flaws in widely used self-hosted LLM platforms. Below is a list of the CVEs assigned to our findings, along with their CVSS severity ratings.

CVE ID Severity Description
CVE-2025-56157 Critical Dify versions up to and including 1.5.1 contain a critical security vulnerability where the PostgreSQL database is configured with hardcoded default credentials in the docker/docker-compose.yaml file. If the PostgreSQL port (5432) is exposed to the internet or an untrusted network, an attacker can gain unauthorized access to the database, leading to sensitive data leakage, privilege escalation, and potentially remote code execution.
CVE-2025-63389 Critical Authentication Bypass in Ollama API. Ollama versions up to v0.1.23 contain a critical vulnerability allowing unauthorized attackers to bypass authentication mechanism in the API, leading to potential code execution and privilege escalation.
CVE-2025-63390 High Authentication Bypass in AnythingLLM Workspaces. A logic flaw in AnythingLLM (v1.8.5) allows attackers to bypass authentication checks for workspace access, resulting in unauthorized information disclosure and privilege escalation.
CVE-2025-63391 Medium Authentication Bypass in Open-WebUI Config Endpoint. Open-WebUI (v0.6.32) fails to properly enforce authentication on configuration endpoints, allowing unauthenticated users to modify system settings.
CVE-2025-63386 Medium CORS Misconfiguration in Dify. Weak Cross-Origin Resource Sharing (CORS) policy in Dify (v1.9.1) allows arbitrary domains to access sensitive resources.
CVE-2025-63388 Medium CORS Misconfiguration in Dify System Features Endpoint. Incorrect CORS configuration exposes system feature endpoints to cross-site attacks in Dify (v1.9.1).
CVE-2025-63387 Medium Unauthenticated Access to System Features in Dify. Lack of access control on specific system features in Dify (v1.9.1) allows unauthenticated attackers to invoke sensitive system functions.

Ethical Disclosure: Note that we reported first to institutional authorities and affected vendors, then to CVE after awaiting remediation without adequate response.
